# 감각소 서비스화 — Context Notes

작업 중 내린 결정과 이유를 기록한다. 다음 세션이 같은 고민을 반복하지 않도록 하는 것이 목적이다.

## 수익 모델
- **애드센스로 시작하기로 결정.** 감각소는 클라이언트 사이드(브라우저 내 JSZip 처리) 구조라 서버가 없다. 그래서 사용량 추적·과금 게이트를 걸기 어렵다. 후원형/서버 크레딧/기능 잠금도 검토했으나, 사용자 기반이 없는 초기 단계에서는 서버·결제 시스템이 필요 없는 애드센스가 가장 단순하다고 판단했다.
- 애드센스는 트래픽이 수익을 좌우한다. 승인에는 독립 도메인 + 일정량의 페이지가 필요해서, 소개/사용법 페이지를 먼저 만들었다.

## 호스팅: 왜 Vercel인가
- 처음엔 GitHub Pages로 진행했으나 가비아에서 도메인 연결(A 레코드/CNAME 입력)이 계속 막혀 Vercel로 전환했다.
- 가비아 웹호스팅은 월 2,000~3,000원 유료. Vercel은 정적 사이트 무료. 도메인만 가비아에서 사고 호스팅은 Vercel로 쓰는 구조가 비용·성능 면에서 합리적이라고 판단했다.

## DNS: 왜 Cloudflare로 우회했나
- 가비아 DNS 호스트 설정이 제약이 많았다. `@` 호스트 입력 불가, CNAME 값에 도메인 형식 입력 불가(IP만 허용), 같은 호스트 중복 불가 등으로 Vercel이 요구하는 레코드를 넣을 수 없었다.
- 해결책으로 가비아 네임서버를 Cloudflare로 변경하고, DNS 레코드 관리를 Cloudflare에서 하기로 했다. Cloudflare는 무료이며 `@`/CNAME 설정이 자유롭다.

## SSL / 프록시 설정 — 시행착오 기록
- Cloudflare 프록시(주황 구름) + SSL 모드 **Flexible** 조합에서 리디렉션 루프가 발생했다.
- SSL 모드를 **Full** 로 바꾸고, "Always Use HTTPS"를 켜서 안정화했다.
- 중간에 프록시를 DNS-only(회색 구름)로 뒀더니 SSL 인증서가 발급되지 않아 접속이 불안정했다. **최종적으로 프록시는 켜고(Proxied) SSL은 Full** 로 두는 것이 정답이었다.
- 참고: Proxied 상태에서는 외부 DNS 조회 시 CNAME이 숨겨지고 Cloudflare 프록시 IP가 응답된다(104.21.x, 172.67.x). 이는 정상 동작이다.

## "사이트 접속 안 됨" 문제의 진짜 원인
- 작업 PC에서만 `DNS_PROBE_FINISHED_NXDOMAIN` 이 떴다.
- 진단 결과: DNS 자체는 전 세계에서 정상(dnschecker 전부 초록), 구글 DNS(8.8.8.8)로도 정상 응답. **작업 PC가 쓰는 통신사(하나로, bns1.hananet.net) DNS만 응답 실패**.
- 휴대폰 LTE에서는 정상 접속됨 → 서비스는 정상이고 특정 통신사 DNS 캐시만 문제라는 것이 확정됐다.
- 해결: 기다리면 갱신되거나, PC DNS를 8.8.8.8 또는 1.1.1.1로 변경하면 즉시 해결. 사용자가 프라이버시 우려로 변경은 보류, 외부 접속이 되는 것으로 서비스 정상을 확인했다.

## 디자인 일관성 원칙
- 활자소에서 이어온 톤 유지: 종이 질감 배경, 제목은 Gowun Batang(세리프), 본문은 IBM Plex Sans KR, 라벨·코드는 IBM Plex Mono, accent는 붉은색(#c0432a).
- 세 페이지 공통 타이포 기준을 통일: 본문 15px/weight 400/line-height 1.75, 보조 설명 14px, 소제목 weight 600, 약한 설명 weight 300.
- 긴 설명 문장은 마침표 뒤에서 `<br>` 로 줄바꿈해 가독성을 맞췄다.

## OG(대표 이미지) 설정
- 대표 주소는 `www.gamgakso.com` 으로 통일.
- OG 이미지는 1200×630, 파일명 `og-image.png`, 사이트 루트에 위치. HTML에서 절대경로(`https://www.gamgakso.com/og-image.png`)로 참조한다.
- 이미지는 사이트 톤과 맞춤(종이 질감, 세리프 타이틀, 붉은 라벨). 초기 버전의 타이틀 하단 장식선은 어색해서 제거했다.
- 카카오/페이스북은 미리보기를 캐시하므로, 변경 시 각 플랫폼의 캐시 초기화 도구로 갱신해야 한다.
